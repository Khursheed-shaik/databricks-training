# Day 1 - SQL Practice and Database Concepts

## What I Learned

Today I worked extensively on SQL fundamentals and intermediate database concepts using Employee, Department, and Project tables. I learned how relational databases are structured and how different tables are connected using keys and relationships.

Through solving multiple SQL queries, I practiced retrieving, filtering, sorting, grouping, and analyzing data using different SQL clauses and functions. I also understood how to work with multiple tables using JOIN operations and how nested queries can be used for solving complex problems.

The practice helped me improve my understanding of writing optimized and readable SQL queries while exploring real-world database scenarios.

## Topics Covered

* SELECT statements for retrieving data
* Filtering records using WHERE conditions
* Pattern matching using LIKE operator
* Working with date functions and conditions
* Aggregate functions such as SUM, AVG, MIN, MAX, and COUNT
* GROUP BY and HAVING clauses for data analysis
* Sorting records using ORDER BY
* INNER JOIN and LEFT JOIN operations
* Nested and subqueries
* Department-wise and employee-wise data analysis
* Salary and experience-based filtering and comparisons

## Work Done

* Executed `datacreation.sql` to create and populate database tables
* Solved SQL queries from Q1 to Q65
* Practiced queries on Employee, Department, and Project tables
* Performed filtering, grouping, sorting, aggregation, and join operations
* Explored advanced SQL concepts using nested queries and subqueries
* Verified outputs using DB-Fiddle
* Organized SQL files, outputs, and documentation in GitHub repository

## Challenges Faced

* Understanding relationships between multiple tables initially
* Writing JOIN queries correctly with matching columns
* Using GROUP BY and HAVING together for aggregate analysis
* Understanding nested queries and subquery logic

## Key Takeaway

This practice strengthened my SQL fundamentals and improved my confidence in solving database-related problems. I gained practical experience in handling relational data, analyzing records using SQL queries, and understanding how different SQL concepts work together in real-world scenarios.
